I am building a Flutter Android app — a personal AI assistant chatbot that can set alarms and calendar reminders on the user's device by understanding natural language messages.

Build the complete AlarmManager alarm system for this app. Here is everything you need to know:

---

PROJECT CONTEXT:
- Flutter app targeting Android only
- The backend server returns a JSON response after every message like this:
  { "reply": "Sure! Alarm set for 6:30 AM.", "intent": { "type": "alarm", "hour": 6, "minute": 30, "label": "Wake up", "datetime_ms": 1735452600000 } }
  or { "reply": "...", "intent": null } for normal chat
- Local SQLite database already exists using sqflite. The intents table schema is:
  CREATE TABLE intents (id INTEGER PRIMARY KEY AUTOINCREMENT, type TEXT NOT NULL, label TEXT, scheduled_at INTEGER NOT NULL, created_at INTEGER NOT NULL, status TEXT DEFAULT 'active', message_id INTEGER, alarm_id INTEGER)
- The app has 4 screens: ChatScreen, ScheduleScreen, MemoryScreen, SettingsScreen

   Register the AlarmBroadcastReceiver, a BootReceiver, the AlarmService background service, and an AlarmActivity with showOnLockScreen=true and turnScreenOn=true

3. lib/services/alarm_service.dart
   A class AlarmService with these methods:
   - static Future<void> initialize() — call AndroidAlarmManager.initialize()
   - Future<int> scheduleAlarm(int hour, int minute, String label, int datetimeMs) — generates a unique int alarm ID, calls AndroidAlarmManager.oneShotAt() with the datetime, saves to SQLite intents table with status 'active', returns the alarm ID
   - Future<void> cancelAlarm(int alarmId) — calls AndroidAlarmManager.cancel(alarmId), updates SQLite status to 'cancelled'
   - Future<void> rescheduleAllOnBoot() — queries SQLite for all intents where status='active' and scheduled_at > DateTime.now().millisecondsSinceEpoch, re-registers each one with AndroidAlarmManager
   - static Future<void> onAlarmFired(int id) — the top-level callback function required by android_alarm_manager_plus, must be a top-level function not a class method. It should look up the alarm in SQLite by alarm_id, then launch the AlarmScreen activity via a local notification using flutter_local_notifications with fullScreenIntent: true

4. lib/services/intent_parser.dart
   A class IntentParser with:
   - static IntentModel? parse(Map<String, dynamic> serverResponse) — checks if serverResponse['intent'] is not null, parses the type, hour, minute, label, datetime_ms fields, returns an IntentModel or null

5. lib/models/intent_model.dart
   A simple data class IntentModel with fields: String type, String label, int hour, int minute, int datetimeMs

6. lib/screens/alarm_screen.dart

IMPORTANT IMPLEMENTATION NOTES:
- The onAlarmFired callback passed to AndroidAlarmManager.oneShotAt() MUST be a top-level Dart function, not a class method or lambda. This is a hard requirement of android_alarm_manager_plus.
- On Android 12+ SCHEDULE_EXACT_ALARM requires sending the user to Settings. Do not use Permission.scheduleExactAlarm.request() — use openAppSettings() instead and let the user toggle it manually.
- The AlarmScreen must be able to display over the lock screen. Set the window flags FLAG_KEEP_SCREEN_ON, FLAG_SHOW_WHEN_LOCKED, FLAG_TURN_SCREEN_ON in the Activity onCreate in MainActivity.kt or via a MethodChannel from Flutter.
- All database operations must be async and use await. Never call sqflite synchronously.
- Generate unique alarm IDs using DateTime.now().millisecondsSinceEpoch % 100000 as an int — AlarmManager requires int IDs not UUIDs.
